# Global Rules

## Git Commit Messages

Follow [Chris Beams' seven rules](https://chris.beams.io/posts/git-commit/) for all commit messages:

1. Separate subject from body with a blank line
2. Limit the subject line to 50 characters (72 hard limit)
3. Capitalize the subject line
4. Do not end the subject line with a period
5. Use the imperative mood in the subject line ("Fix bug", not "Fixed bug" or "Fixes bug")
6. Wrap the body at 72 characters
7. Use the body to explain *what* and *why* vs. *how*

**Never use conventional commit prefixes** (feat:, chore:, fix:, docs:, etc.). Write plain English subject lines.

The subject line should complete this sentence: "If applied, this commit will..."
